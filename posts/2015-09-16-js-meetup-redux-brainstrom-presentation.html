JS meetup presentation on prototyping with redux

Live coding with redux and hot code swap.

When asked to give a talk a couple days before the JS meetup in Shanghai I decided that saying "maybe next time" for the fourth (fifth?) time would be too lame and that I needed to man up and find the time to prepare.

I had long since had the idea of making a brainstorming tool, a way to add a concept and be able to connect it to others.

I also wanted to demonstrate how nice JavaScript is becoming with newly emerging tools and (light) frameworks. The topic "rapid prototyping with short development cycle" was coined. On the mobile SPA overhaul of the data.worldbank.org redux was chosen and it has been great and pretty stable, increasing development speed (and thus joy).

I talked about how the following:

  • Running through the set-up steps to get a hotloading react/redux project.
  • Setting up a basic store and reducer function for listening to actions that modify the graph state (a brainstorm perfectly fits a graph!).
  • Adding actions for adding/removing/merging and moving concepts.
  • Switching to immutable.js to easily compare current app state vs the wanted state to look like and add React components that visualize these data structures.
  • Added a d3 force directed graph that displays the concepts as bubbles.
  • Ran out of time writing the recursive-reducing function that collects the links (edges) between concepts (nodes).
I prepared a quick demo of the process and uploaded the slides and different steps to the branches of a github repo

You can follow along the slides (and switch branches accordingly) if you'd like to get an impression of that the next JavaScript standard will be like with hot code swapping.

I'd like to continue developing this app and add things like textareas, selecting bubbles and drag and dropping to merge them. Later also adding collaboration features through time constrained "phases" that allow to vote for merging/expanding/finishing eachothers mindmaps in a democratic way. Brainstorming is usually a group activity after all!

Maybe that might be cool to do while learning ClojureScripts Re-Frame framework.
